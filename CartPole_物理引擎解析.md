# CartPole 物理引擎解析 —— gym.step() 背后的 World Model

## 1. 什么是 CartPole（倒立摆）

想象一根杆子竖直插在一辆小车顶上，通过一个铰链（可以自由旋转的关节）连接。杆子随时可能倒下，你唯一能做的就是推小车——向左或向右。

```
        |        ← 杆（会倒）
        |
     ===●===     ← 小车（你可以推它）
  ________________   ← 轨道（有限长度）
```

**目标**：通过左右推小车，让杆尽可能久地保持直立。

这就是经典的 **倒立摆控制问题（Inverted Pendulum）**，也是强化学习最常用的入门环境之一。

---

## 2. 四个状态变量的物理含义

CartPole 的状态由 4 个数字完整描述：

| 变量 | 符号 | 含义 | 单位 | 范围 |
|------|------|------|------|------|
| 小车位置 | `x` | 小车在轨道上的位置，0 = 中心 | 米 (m) | 出界条件: \|x\| > 2.4 |
| 小车速度 | `x_dot` (v) | 小车移动的速度，正 = 向右 | 米/秒 (m/s) | 无限制 |
| 杆角度 | `theta` (θ) | 杆偏离竖直方向的角度，正 = 向右倒 | 弧度 (rad) | 倒下条件: \|θ\| > 12° |
| 杆角速度 | `theta_dot` (ω) | 杆旋转的速度 | 弧度/秒 (rad/s) | 无限制 |

**关键洞察**：仅仅知道角度 θ 是不够的！你还需要知道角速度 ω——杆正在向哪个方向倒、倒得有多快。同理，知道小车位置 x 不够，还需要速度 v。

这就是为什么状态是 4 维而不是 2 维。**位置决定了"在哪"，速度决定了"要往哪去"。**

---

## 3. 物理方程：力 → 加速度 → 速度 → 位置

### 3.1 因果链

整个物理引擎的核心逻辑是一条因果链：

```
动作 (action)
  ↓
力 (force = ±10N)
  ↓
加速度 (牛顿第二定律: F = ma → a = F/m)
  ↓
速度 (v_new = v_old + a × dt)
  ↓
位置 (x_new = x_old + v × dt)
```

但 CartPole 比"推一个箱子"复杂，因为**杆和小车是耦合的**——推小车时杆会晃，杆晃动又会影响小车。

### 3.2 物理常数

```python
gravity  = 9.8    # 重力加速度 (m/s²)
m_cart   = 1.0    # 小车质量 (kg)
m_pole   = 0.1    # 杆的质量 (kg)
M_total  = 1.1    # 总质量 (kg)
L        = 0.5    # 杆的半长 (m)
F        = 10.0   # 推力 (N)
dt       = 0.02   # 时间步长 (s)
```

### 3.3 核心方程（简化解读）

这些方程来自拉格朗日力学（Lagrangian mechanics），是对牛顿定律在约束系统下的推广。

**第一步：中间变量 temp**

```
temp = (F + m_pole × L × ω² × sin(θ)) / M_total
```

- `F` 是你施加的推力
- `m_pole × L × ω² × sin(θ)` 是杆旋转产生的离心力的水平分量
- 除以总质量做归一化
- 可以理解为"小车的粗略加速度"

**第二步：杆的角加速度 θ̈**

```
        g × sin(θ) - cos(θ) × temp
θ̈ = ─────────────────────────────────────
      L × (4/3 - m_pole × cos²(θ) / M_total)
```

分子的物理含义：
- `g × sin(θ)` → 重力让杆倒下的力矩。杆越倾斜，这个力越大
- `- cos(θ) × temp` → 小车加速度通过铰链传给杆的反作用力矩

分母的物理含义：
- `L × (4/3 - ...)` → 杆的有效转动惯量
- `4/3` 来自均匀细杆绕端点旋转的转动惯量公式 `I = (1/3)mL²`

**第三步：小车的加速度 ẍ**

```
ẍ = temp - m_pole × L × θ̈ × cos(θ) / M_total
```

- 在粗略加速度 `temp` 基础上，减去杆的角加速度对小车的反作用

**注意耦合**：θ̈ 的计算用到了 temp，ẍ 又用到了 θ̈。杆和小车的运动互相影响，不能独立求解。

### 3.4 欧拉积分

有了加速度，用最简单的数值积分（欧拉法）更新状态：

```
x_new     = x     + dt × v        # 新位置 = 旧位置 + 速度 × 时间
v_new     = v     + dt × ẍ        # 新速度 = 旧速度 + 加速度 × 时间
θ_new     = θ     + dt × ω        # 新角度 = 旧角度 + 角速度 × 时间
ω_new     = ω     + dt × θ̈       # 新角速度 = 旧角速度 + 角加速度 × 时间
```

每一步模拟 0.02 秒，即 1 秒 = 50 步。

### 3.5 完整的状态转移

把上面串起来，就是 CartPole 的 **world model**：

```
输入: (x, v, θ, ω, action)
  ↓
F = +10N 或 -10N              # 动作转换为力
  ↓
temp = (F + 离心力) / 总质量    # 中间变量
  ↓
θ̈ = (重力力矩 - 反作用力矩) / 转动惯量   # 杆的角加速度
  ↓
ẍ = temp - 杆的反作用 / 总质量            # 小车的加速度
  ↓
欧拉积分: 加速度 → 速度 → 位置
  ↓
输出: (x_new, v_new, θ_new, ω_new)
```

**这就是 `gym.step(action)` 在内部做的全部事情。**

## 4. 为什么这是一个 "World Model"

在强化学习中，**world model（世界模型）** 是一个函数：

```
next_state, reward = world_model(current_state, action)
```

CartPole 的世界模型就是上面这套物理方程。它的特点：

1. **确定性**：给定相同的 (state, action)，永远产生相同的 next_state
2. **完全已知**：所有方程和参数都是公开的
3. **可微分**：理论上可以求梯度（虽然 DQN 不需要）
4. **低维**：只有 4 个状态变量和 2 个动作

有了 world model，理论上你可以直接用**规划（planning）**来找最优策略——不需要学习，直接模拟所有可能的动作序列，选最好的。

但 DQN 偏偏**不利用** world model。它把环境当作黑盒：
- 输入 action
- 观察 next_state 和 reward
- 通过试错学习 Q 函数

这是**model-free**（无模型）方法的核心思想。

## 5. 和 Atari DQN 的对比

| 对比维度 | CartPole | Atari (如 Breakout) |
|----------|----------|---------------------|
| **状态** | 4 个数字 (x, v, θ, ω) | 210×160×3 的像素图像 |
| **状态维度** | 4 | ~100,000 |
| **World Model** | 已知的物理方程 | 不存在（只有模拟器黑盒） |
| **物理引擎** | 牛顿力学，可以写出方程 | 游戏内部逻辑，无法用方程描述 |
| **能否规划** | 可以（因为 model 已知） | 不行（model 未知） |
| **为什么用 DQN** | 教学目的（其实不需要） | 真正必要（无法用物理方程求解） |

### 关键区别

**CartPole**：
```
world_model = 物理方程 (可以写出来)
→ 理论上可以用 model-based 方法（如 MPC、LQR）直接求解
→ DQN 在这里是"杀鸡用牛刀"，但很适合教学
```

**Atari**：
```
world_model = 游戏模拟器 (二进制代码，无法分析)
→ 只能通过试错学习（model-free）
→ DQN 在这里才是真正的刚需
```

### DQN 的通用性

DQN 的强大之处在于：**同一套算法，既能解 CartPole（4 维物理），也能解 Atari（10 万维像素）。**

它不需要知道物理方程，不需要知道游戏规则。它只需要：
1. 状态（观测）
2. 动作（选择）
3. 奖励（反馈）
4. 下一个状态

这就是 model-free 方法的魅力——**通用性换效率**。代价是需要大量试错。
