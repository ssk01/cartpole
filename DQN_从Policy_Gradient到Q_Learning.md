# 从 Policy Gradient 到 DQN：给会写 Pong 的人看的 Q-Learning 指南

---

## 你已经会什么

假设你已经跟着 Karpathy 的 "Pong from Pixels" 博客实现过 policy gradient。你知道：

- 网络输入是游戏画面像素，输出是"上移的概率"
- 玩完一局，拿到最终结果（赢 +1 / 输 -1）
- 用这个结果去**加强赢局中做过的动作、削弱输局中做过的动作**
- 核心公式：`gradient = log_prob(action) × reward`
- 然后梯度上升更新网络

这个方法叫 **Policy Gradient**——直接学"看到画面该做什么动作"。

---

## DQN 走了一条完全不同的路

Policy Gradient 的思路是：**直接学策略（policy）**——看到状态 s，输出动作概率。

DQN 的思路是：**先学价值（value），再从价值推导出策略**。

什么意思？

### "价值"是什么？

想象你在打 Pong，球刚从你的拍子上弹出去，飞向对手。这个时刻，你心里大概有个感觉："这球打得不错，对手可能接不住，我**大概率**要得一分。"

这个"大概率要得一分"的感觉，就是**价值（value）**。

更精确地说：

> **Q(s, a) = 在状态 s 下执行动作 a 之后，从现在到游戏结束，我预期还能拿到多少分？**

三个关键词：
- **"还能"**——只算未来的，已经拿到的分数和 Q 值无关。你已经得了 100 分还是 0 分，不影响当前的 Q 值。Q 只看前方，不看过去。
- **"预期"**——不是确定能拿到，是统计意义上的期望值。
- **"总共"**——不是下一步能拿多少（那是 reward），而是从现在到结束所有未来奖励的折扣总和。

举个 Pong 的例子：

```
状态 s：球在中场飞向对手，对手在错误的位置
动作 a：不动（不需要动，球已经打出去了）

Q(s, a) ≈ +0.8
意思是：在这种状态下什么都不做，我预期最终能拿到 +0.8 分
（不是满分 1.0，因为对手有小概率接住球）
```

```
状态 s：球飞向你，你的拍子在屏幕另一端
动作 a：向球移动

Q(s, a) ≈ -0.3
意思是：虽然我在往对的方向跑，但可能来不及，预期结果不太好
```

### 有了 Q 值，怎么做决策？

**太简单了——每次选 Q 值最大的动作就行。**

```python
# Policy Gradient 的做法：
action = sample_from(network(state))  # 按概率采样

# DQN 的做法：
q_values = network(state)  # 输出每个动作的 Q 值
action = argmax(q_values)  # 选最大的那个
```

这就是两种方法最根本的区别：
- **Policy Gradient**：网络输出**概率**，按概率随机选
- **DQN**：网络输出**价值**，选价值最高的（确定性的）

---

## 那 Q 值怎么学？

这是关键问题。Policy Gradient 学策略的方式很直觉——赢了就加强，输了就削弱。Q 值怎么学？

### 核心想法：Bellman 方程

有一个非常优美的递推关系：

> **Q(s, a) = 这一步的即时奖励 + 下一步能拿到的最好价值**

用公式写：

$$Q(s, a) = r + \gamma \times \max_{a'} Q(s', a')$$

其中：
- $r$ = 这一步拿到的即时奖励（如 Pong 里得分 +1 或 0）
- $s'$ = 执行动作后到达的下一个状态
- $\max_{a'} Q(s', a')$ = 在下一个状态里，选最好的动作能拿到的价值
- $\gamma = 0.99$ = 折扣因子，表示"未来的奖励比现在的稍微不值钱一点"

### 用 Pong 走一遍

```
帧 100：球从拍子弹出，飞向对手
  状态 s_100, 动作 "不动", 奖励 r=0（球还在飞，没得分）
  
  Q(s_100, 不动) 应该等于多少？
  = 0 + 0.99 × max Q(s_101, ?)
  = 0.99 × 下一帧最好的 Q 值

帧 150：对手没接住！得分了！
  状态 s_150, 奖励 r=+1
  
  Q(s_150, ?) = 1 + 0  （游戏这一分结束了，没有下一帧）

问题是：帧 100 怎么知道帧 150 要得分？
```

答案：**不需要一次学会，Q 值会从后往前慢慢传播。**

```
第 1 轮训练：帧 150 学到 Q ≈ 1.0（因为直接拿到了奖励）
第 2 轮训练：帧 149 看到下一帧(150)的 Q=1.0 → 自己的 Q = 0 + 0.99×1.0 = 0.99
第 3 轮训练：帧 148 看到下一帧(149)的 Q=0.99 → 自己的 Q = 0 + 0.99×0.99 = 0.98
...
第 50 轮训练：帧 100 终于也学到了 Q ≈ 0.6（价值从帧 150 一路传回来了）
```

这就是为什么 DQN 需要**大量训练**——Q 值需要一步步回传。

## 对比：两种方法怎么学习

### Policy Gradient（你已经会的）

```
1. 玩完一整局
2. 赢了 → 这局里所有动作的概率都增加
   输了 → 这局里所有动作的概率都降低
3. 更新网络
4. 回到 1
```

**优点**：简单直觉，代码少
**缺点**：必须等一局结束才能学；信号噪声大（赢的局里也有烂动作）

### DQN

```
1. 走一步，存下来 (状态, 动作, 奖励, 下一状态)
2. 从历史存储的经验里随机抽 32 条
3. 对每条计算"Q 值应该是多少"（用 Bellman 方程）
4. 让网络的 Q 值预测接近这个目标（普通的监督学习 loss）
5. 反向传播更新网络
6. 回到 1
```

**优点**：每一步都在学（不用等一局结束）；数据可以反复用
**缺点**：需要额外的工程组件（经验回放、目标网络）

---

## DQN 比 Policy Gradient 多出来的"零件"

### 零件 1：经验回放缓冲区（Experience Replay）

Policy Gradient 用完数据就扔。DQN 把所有经验存在一个大数组里（100 万条），训练时随机抽样。

**为什么需要？** 两个原因：

1. **打破时间相关性**。连续的帧几乎一样——如果你用帧 1、2、3、4 连续训练，网络会过拟合到这段连续数据。随机抽样打乱顺序，每个 batch 里混着不同时间、不同局的经验。

2. **数据复用**。一条经验可以被抽到多次，比 Policy Gradient 的"用完即扔"高效得多。

```python
# 你已经熟悉的 Policy Gradient：
play_one_episode()
update_network(episode_data)
throw_away(episode_data)  # 数据用完就扔

# DQN：
for each_step:
    buffer.store(experience)         # 存下来
    batch = buffer.random_sample(32) # 从历史随机抽
    update_network(batch)            # 训练（这些数据以后还能被抽到）
```

### 零件 2：目标网络（Target Network）

这是 2015 Nature 版新加的。问题是什么？

回忆 Bellman 方程：

$$Q(s, a) = r + \gamma \times \max_{a'} Q(s', a')$$

等号左边的 Q 是网络**要学的值**，等号右边的 Q 也是**同一个网络**算出来的。每次你更新网络让左边更准，右边也跟着变了——**靶子在移动**。

解决方案：弄两个网络：
- **策略网络（policy net）**：正常训练，每步更新
- **目标网络（target net）**：参数冻结，只用来算等号右边的 Q 值

每隔 10000 步，把策略网络的参数复制给目标网络。这样在这 10000 步之间，靶子是固定的。

```python
# 没有目标网络（2013 版，靶子会动）：
target = r + γ × max(policy_net(next_state))  # ← 这个网络刚被更新过，不稳定
loss = (policy_net(state) - target)²

# 有目标网络（2015 版，靶子固定）：
target = r + γ × max(target_net(next_state))  # ← 这个网络参数冻住了，稳定
loss = (policy_net(state) - target)²

# 每 10000 步同步一次：
target_net.weights = policy_net.weights
```

### 零件 3：ε-greedy 探索

Policy Gradient 天然有探索性——它输出概率，按概率采样，所以总会偶尔尝试不同的动作。

DQN 输出的是确定性的 Q 值，直接 argmax 选最大的——**完全没有探索**。如果一开始某个动作碰巧 Q 值高一点，它就会一直选这个动作，永远不知道别的动作可能更好。

解决方案很简单：ε-greedy。

```python
if random() < ε:
    action = 随机选一个   # 探索（ε 的概率）
else:
    action = argmax(Q)   # 利用（1-ε 的概率）
```

ε 从 1.0（完全随机）慢慢衰减到 0.01（几乎不随机），让 agent 从"到处乱试"过渡到"基本上选最好的"。

## 网络输入输出的对比

```
Policy Gradient（Karpathy Pong）：
  输入：80×80 像素 → 网络 → 输出：P(上移) = 0.7
  含义："看到这个画面，有 70% 概率应该按上"

DQN（Atari）：
  输入：84×84×4 堆叠帧 → 网络 → 输出：[Q(上)=3.2, Q(下)=1.1, Q(左)=0.5, ...]
  含义："看到这几帧画面，上移能拿 3.2 分，下移能拿 1.1 分……选上移"
```

注意 DQN 的输出不是概率，是**预期总回报的估计值**。数值本身可以是任意实数，不需要加和为 1。

---

## 训练信号的对比

这是最本质的区别：

```
Policy Gradient：
  信号 = log π(a|s) × R
  R 来自哪里？玩完一整局才知道。
  信号质量：噪声大（赢的局里也有臭棋）

DQN：
  信号 = [r + γ max Q(s') - Q(s,a)]²
  r 来自哪里？每一步就有。
  Q(s') 来自哪里？网络自己估计的。
  信号质量：也有误差（Q 估计可能不准），但方差低得多
```

Policy Gradient 的信号质量差但很"纯粹"——直接来自环境的真实结果。
DQN 的信号质量好但是**自举的**（bootstrapped）——用自己的估计来训练自己。这就是为什么需要目标网络来稳定。

---

## DQN 的 Reward 机制

### Atari 的 reward 不是一局结束才给

和 Policy Gradient 打 Pong（一局结束才有 +1/-1）不同，DQN 的 reward 是**每一帧都有的**——每帧的分数变化就是 reward。打掉砖块 → +1，什么都没发生 → 0，丢命 → -1。

但**延迟奖励**仍然存在：好的动作可能要很多帧以后才能拿到分。Pong 里你挥拍（帧 100）到对手漏接（帧 150）中间隔了 50 帧。帧 100 的 reward 是 0，网络怎么知道这是好的动作？

靠 Q 值的**逐步回传**。Bellman 更新每次只传一步：帧 150 的 Q 值先涨（因为直接拿到 reward），然后帧 149 看到下一帧 Q 值高了也跟着涨，一步步传回帧 100。延迟 50 帧就需要至少 50 轮更新才能传到——这就是为什么 DQN 要训练 1000 万帧。

### 经验回放缓冲区怎么淘汰？

**FIFO 先进先出**。固定 100 万条，满了最老的被覆盖（环形数组）。不区分重要性——关键得分经验和无聊飞行经验被抽到的概率一样。后来的**优先经验回放**（2016）解决了这个问题。

### 每个游戏独立训练

7 个游戏 = 7 次训练 = 7 组权重。缓冲区里只有当前游戏的数据。"同一套架构和超参数"是指配置不针对每个游戏调参，但训练过程分开——Pong 的网络不会打 Breakout。

## DQN 的 Loss：为什么在"自己猜自己"？

这是 DQN 最诡异也最核心的地方。

### 监督学习的 loss

```
loss = (网络预测 - 人类标注的正确答案)²
                    ↑ 外部给的真相，固定不变
```

### DQN 的 loss

```
loss = (Q(s,a; θ) - [r + γ × max Q(s'; θ⁻)])²
                    ↑ 网络自己（的旧版本）算出来的！
```

训练网络 A 去逼近一个目标，但目标是网络 B（target net）算的，B 就是 A 的旧版本。**用自己的猜测训练自己**——这叫**自举（bootstrapping）**。

### 为什么这能 work？

Bellman 方程有一个数学上的**不动点**。虽然每一步的 target 不精确，但方向是对的。就像一群盲人排队，第一个摸到墙（reward），告诉第二个"墙在那边"，第二个告诉第三个……最终所有人都知道墙在哪。

```
第 1 轮：Q(终点) 因为直接拿到 reward 变成 1.0
第 2 轮：Q(倒数第二步) = 0 + 0.99 × 1.0 = 0.99
第 3 轮：Q(倒数第三步) = 0 + 0.99 × 0.99 = 0.98
...
Q 值从 reward 发生处一步步向前传播
```

### 为什么需要 target network？

没有 target network 时，左边和右边用同一个 θ：更新 θ 让左边更接近右边，但右边也跟着变了——**追着自己尾巴跑**，可能正反馈导致 Q 值爆炸。

Target network 冻住右边的 θ⁻，在 C 步之内目标是固定的——变成了一个正常的"临时监督学习"问题。

### Loss 曲线怎么看？

DQN 的 loss 曲线和监督学习不同——不是单调下降的：

- **开始 loss ≈ 0**：Q 值全接近 0，目标也接近 0，差距小——不是学得好，是还没开始学
- **loss 飙升**：网络开始学到某些状态有价值，目标变大但 Q(s,a) 没跟上——这是好事
- **loss 下降并稳定**：Q(s,a) 追上了目标，但永远不会到 0（目标一直在变）

DQN 论文不看 loss，看**平均 Q 值**和**平均回报**——loss 在 RL 中不是好的进度指标。

### 和 Policy Gradient 的核心 trade-off

- **Policy Gradient**：信号来自环境真实结果——真实但方差极大（赢的局里也有臭棋全被加强）
- **DQN**：信号来自自己的估计——方差小但有偏（自己猜自己，可能系统性偏差）

**真实但方差大 vs 有偏但方差小**——这个 trade-off 贯穿整个 RL 领域。

## 代码关键行解读

### gather vs max：回忆过去 vs 假设未来

```python
# 第一行：取"已经选了的"那个动作的 Q 值
q_values = policy_net(states_t).gather(1, actions_t.unsqueeze(1)).squeeze(1)

# 第二行：取"最好的"动作的 Q 值
next_q_max = target_net(next_states_t).max(1)[0]
```

用具体数字走一遍：

```
policy_net 输出：[Q(左)=3.2, Q(右)=1.1]
actions_t = [0]        ← 当时选了左
.gather → 取 3.2       ← "当时选的那个动作，现在估计值多少"

target_net 输出：[Q(左)=4.1, Q(右)=3.5]
.max → 取 4.1           ← "下一步最好能拿多少"
```

**gather 是回忆过去的选择**（经验里记录了"当时选了动作 a"，不能改）
**max 是假设未来最优**（Bellman 方程说"下一步你会选最好的"）

合在一起：

```
loss = (已选动作的 Q 值 - [即时奖励 + γ × 未来最好的 Q 值])²
        ↑ gather，policy_net      ↑ max，target_net（冻住的）
```

一句话理解 DQN 的核心：**拿当前 Q 值去拟合 Bellman 方程算出的目标值。**

### 帧跳和帧堆叠

DQN **每 4 帧才做一次决策**（帧跳 k=4），中间重复上一个动作——省计算。

输入是**最近 4 个决策帧堆叠**（84×84×4），不是连续 4 帧——实际跨越 16 个原始帧。堆叠是为了**看到运动**：单帧看不出球往哪飞，4 帧堆叠后网络能从帧间差异推断速度和方向。

CartPole 不需要帧差/帧堆叠——因为 state 里直接给了速度和角速度（state[1] 和 state[3]）。这 4 个数字相当于已经做完特征提取的结果。

## Demo 代码导读

`dqn_demo.py` 的结构对应上面的概念：

```
QNetwork        ← 就是网络，输入状态，输出每个动作的 Q 值
ReplayBuffer    ← 经验回放缓冲区，存数据、随机抽数据
select_action   ← ε-greedy 策略
train_step      ← 核心：从缓冲区抽样 → 算 Bellman 目标 → 算 loss → 反向传播
train()         ← 主循环：每帧 玩一步+存一条+训一步
```

用的是 CartPole（平衡杆）而不是 Atari——原理完全一致，只是输入从像素变成了 4 个数字（杆的角度、角速度、小车位置、小车速度），网络从 CNN 变成了 MLP。这样训练几分钟就能看到效果，而不是等几个小时。

## 一句话总结

| | Policy Gradient | DQN |
|--|----------------|-----|
| 学什么 | 策略：s → P(a) | 价值：s → Q(s,a) |
| 怎么决策 | 按概率采样 | 选 Q 最大的 |
| 训练信号 | 一局的总 reward | 每步的 TD 误差 |
| 数据用法 | 用完即扔 | 存起来反复用 |
| 探索方式 | 天然概率采样 | 需要 ε-greedy |
| 额外组件 | 无 | 回放缓冲区 + 目标网络 |
| 代码量 | ~130 行 | ~200 行 |
| 优势 | 简洁、直觉 | 数据高效、稳定 |
